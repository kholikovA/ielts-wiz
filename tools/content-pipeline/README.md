# Content pipeline

Turns an extracted test spec into a live test, for **reading and listening**. Three
content-agnostic, dependency-free Node scripts (Node 18+). They operate on whatever
spec you give them — sourcing and rights to that content are your responsibility.

```
extract (IELTSX extension)  →  spec.json
        │
        ├─ host-assets.mjs   (listening only: audio + images → your GitHub → URLs in spec)
        ├─ schema.mjs        (validate it will play + grade)
        └─ ingest.mjs        (slug + place file + register in manifest.js)
              │
              └─ (reading only, optional) AI evidence/vocabulary enrichment → PR
```

## 1. Validate — `schema.mjs`
Checks the spec matches what the React players consume (types, gaps vs questions,
answer-key coverage). A spec that passes here will render and grade.

```bash
node tools/content-pipeline/schema.mjs path/to/spec.json listening
```

## 2. Host assets — `host-assets.mjs` (listening)
Downloads the audio + any map/diagram labelling images the extractor captured,
uploads them to your own GitHub Pages repo, and rewrites the spec's `audio_url` and
`image_url`s to the public hosted URLs. Native flowcharts have no image, so nothing
to host there.

```bash
GITHUB_TOKEN=ghp_xxx node tools/content-pipeline/host-assets.mjs spec.json \
  --repo kholikovA/ielts-assets \
  --dir listening/test2 \
  --base-url https://kholikovA.github.io/ielts-assets/listening/test2 \
  --write
```
Run it without `--write` (or without a token) for a dry run that just lists what
would be hosted.

## 3. Ingest — `ingest.mjs`
Validates, mints an opaque URL slug, places the JSON in the right data folder, and
registers it in `src/data/tests/manifest.js`. Dry-run by default.

```bash
# preview
node tools/content-pipeline/ingest.mjs spec.json --skill listening --id iw_listening_2 --source iw_practice --n 2
# do it
node tools/content-pipeline/ingest.mjs spec.json --skill listening --id iw_listening_2 --source iw_practice --n 2 --write
```
The test is then live at `/<skill>-test/<slug>` after a build.

## Reading enrichment (separate, AI)
The structured `explanations` (verbatim evidence + paragraph + rationale) and
`vocabulary` that power the reading review aren't in the source page — they're
generated by an AI pass over each passage, after ingest. Ask for that step when a
reading test is in.

## Per-test workflow
1. Open the test on IELTSX → extract with the extension → download `spec.json`.
2. Listening only: `host-assets.mjs … --write`.
3. `schema.mjs spec.json <skill>` → fix any ✗.
4. `ingest.mjs spec.json … --write`.
5. Reading only: run enrichment. Then open a PR.
